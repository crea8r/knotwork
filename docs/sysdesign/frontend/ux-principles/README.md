# UX Principles

Cross-cutting design decisions that apply across all pages and surfaces.

- **mental-model.md** — The foundational design document. Starts from "what decisions
  is the user forced to make that they shouldn't have to make?" Covers: four-layer
  architecture (Asset / Project / Channel / Agent), two working modes (Strategy /
  Operation), three navigation surfaces (Now / Work / Knowledge), channel architecture,
  chat as parallel mode, runs-belong-to-projects constraint, and the design filter.
  **Read this first.**

- **principles.md** — Design principles derived from the mental model, plus the
  concrete information architecture: navigation model (4 items), full IA tree, route
  map, and channel types.

- **ux-patterns.md** — Reusable interaction patterns: collapsible project dashboard,
  inner sidebar channel list, review queue, unified asset tree, status badges, token
  warnings, health indicators, empty states, modal pattern, mobile behaviour.
