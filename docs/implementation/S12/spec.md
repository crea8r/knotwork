# Session 12 — Workspace Representatives, Agent Zero, and MCP Expansion

## Goal

Clarify how human and agent representatives interact with Knotwork after the project layer exists, and expand MCP so representatives can operate work from their native environment.

S12 is about:

- workspace representatives
- Agent Zero as optional workspace orchestrator
- MCP expansion for project/task operations
- clarifying the OpenClaw plugin vs MCP split

S12 is not the session for project-level AI status writing or project intelligence behavior. Those project-AI interaction questions are deferred to S12.2.

## In Scope

1. Workspace representatives.
   - designate one or more representatives
   - representatives may be human or registered agent identities
2. Agent Zero.
   - optional workspace orchestrator
   - guided onboarding for bootstrapping workspace structure
   - may help create initial projects, handbook content, and agent setup
3. MCP expansion.
   - add project and task operations to the MCP surface
   - support representative-driven structured work from external agent contexts
4. OpenClaw / MCP boundary clarification.
   - OpenClaw plugin is the inbound Knotwork delivery path
   - MCP is the agent -> Knotwork interaction surface
5. Workspace bulletin.
   - one workspace-wide channel for announcements and shared updates

## Explicitly Out of Scope

- AI-authored or AI-drafted project status updates
- autonomous project progress summaries
- objective refinement or project-intelligence workflows as product features
- post-MCP designer/workflow chat agent participation details
- workload-honesty redesign

Those deferred interaction questions belong to S12.2.

## Acceptance Criteria

1. Workspace representatives are a clear first-class concept in product language and architecture.
2. S12 defines the MCP expansion for project/task operations.
3. S12 clarifies the role of MCP vs OpenClaw plugin.
4. Agent Zero is framed as optional workspace orchestration, not as mandatory product intelligence.
5. Project-level AI status/update behavior is explicitly deferred to S12.2.
