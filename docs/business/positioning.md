# Positioning

## Category

Knotwork is a knowledge-driven workflow operating layer for teams.

It turns SOPs, project context, and human review into repeatable execution across humans and agents.

## What Knotwork Is

- a supervised workflow system
- a run and escalation system for team work
- a place where documentation, collaboration, and agent execution meet
- a neutral orchestration layer above Claude, Codex, OpenClaw, or future agent systems

## What Knotwork Is Not

- not an extension of Claude or Codex
- not a Slack replacement
- not a chat-first team messenger
- not an integration-first automation platform like n8n or Zapier
- not a general-purpose coding agent IDE

## Core Differentiation

Knotwork is strongest when work is:

- repeatable
- judgment-heavy
- knowledge-dependent
- review-driven
- expensive to do inconsistently

The product wedge is not "more AI."
The wedge is:

- executable SOPs
- explicit run state
- human checkpoints
- traceable decisions
- improving the way a team works over time

## Short Positioning Options

### Default

Knotwork helps teams run complex work with agents, SOPs, and human review.

### More explicit

Knotwork turns SOPs, project context, and human review into repeatable execution.

### More defensive

Knotwork is the operating layer for knowledge-governed work.

## Positioning Traps To Avoid

### "Claude/Codex extension"

This makes Knotwork look subordinate to execution engines.

### "Slack + agents"

This makes Knotwork look like a chat layer instead of a workflow system.

### "Notion + Slack + Claude"

This is directionally useful, but too broad unless you add "with workflow/runs at the center."

### "AI workflow builder"

This makes Knotwork look like Dify, Flowise, or n8n.

## Recommended One-Liner

Knotwork is where teams turn SOP-driven work into supervised workflows, using agents like Claude, Codex, or OpenClaw as execution engines.
