# Business Docs

This folder holds the core business documents for Knotwork.

These docs are meant to be operational, not aspirational. Each one should help with one of:

- deciding what market to pursue
- deciding what to say
- deciding who to talk to
- deciding what threat matters
- deciding what to test next

## Suggested Reading Order

1. `positioning.md`
2. `icp.md`
3. `market-map.md`
4. `threats-and-moats.md`
5. `messaging.md`
6. `customer-discovery.md`
7. `alpha-program.md`

## Current Product Thesis

Knotwork should be treated as:

- a workflow operating layer for teams
- a system for supervised, knowledge-governed work
- a product that can use Claude, Codex, OpenClaw, or other agents as execution engines

Knotwork should not be framed as:

- an extension of Claude or Codex
- a Slack clone with agents
- a generic integration platform like n8n

## Current Open Questions

- which first industry to focus on
- which channel can reliably reach that industry
- whether the first wedge is compliance-heavy services, recruiting/staffing, or internal operations teams
- how much of the value proposition should lead with workflows vs handbook execution vs human review
