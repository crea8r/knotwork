# Market Map

## Closest Categories

Knotwork overlaps with multiple categories, but is not identical to any one of them.

### 1. Integration / automation tools

Examples:

- n8n
- Zapier
- Relay.app
- Copilot Studio

Where they are strong:

- connectors
- deterministic automation
- existing enterprise buying patterns

Where Knotwork should differ:

- handbook execution
- supervised runs
- escalations as first-class workflow state
- knowledge-governed work rather than app-to-app automation

### 2. Agent builders / orchestration tools

Examples:

- Dify
- Flowise
- LangGraph
- CrewAI

Where they are strong:

- developer flexibility
- agent orchestration primitives
- open-source adoption

Where Knotwork should differ:

- operator UX
- business workflow shell
- workflow state for teams, not just agents

### 3. Agent execution surfaces

Examples:

- Claude
- Codex
- OpenClaw
- Google Antigravity

Where they are strong:

- model quality
- execution environment
- developer mindshare

Where Knotwork should differ:

- execution governance
- cross-agent workflow state
- human checkpoints
- system-of-record for supervised work

## Competitive Summary

### Against n8n

n8n is integration-first.
Knotwork should be workflow-governance-first.

### Against Claude/Codex

Claude and Codex are execution surfaces.
Knotwork should be the operating layer around the work.

### Against Slack-native workflow products

Slack products win on distribution and habit.
Knotwork should win on explicit work state, traceability, and SOP execution.

## Buyer Confusion Risks

Knotwork will be misunderstood if it is described as:

- AI workflow builder
- agent collaboration platform
- team chat with agents

That causes buyers to compare it against stronger incumbents in the wrong category.

## Useful Competitive Sentence

Knotwork is not trying to replace Slack, Claude, or n8n. It is the supervised workflow layer that sits above chat tools, docs, and execution agents.
