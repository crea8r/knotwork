# Assets Module

Scope:
- knowledge
- file and folder flows
- storage adapters
- suggestions
- conversions
- asset health and change flows

Ownership rules:
- asset and document behavior belongs here
- generic storage or shared contracts belong in `libs`
- cross-module coordination belongs in `core`

Docs and tests:
- docs should go in `modules/assets/docs/`
- tests should go in `modules/assets/tests/`
